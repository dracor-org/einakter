## What Is This?

This database is derived from the working corpus for Dîlan's dissertation
on one-act plays (submitted in October 2021). The growing collection is also
intended to provide a general quantitative overview of German-language one-act
plays between the mid-18th and mid-19th centuries.

For each play we list metadata and links to other projects, if available
(editions at Google Books, Wikidata identifiers, DraCor,
Carl-Maria-von-Weber-Gesamtausgabe, etc.). The long-term goal is
to place metadata on one-act plays within the growing knowledge repositories of
the Linked Open Data cloud.

The database was first released in June 2020 and is a work in progress. If you

## Criteria

It is not always easy to decide whether a play should be considered a one-act
play or not. The first German-language plays with the addition »in einem Akt«,
or, »in einem Aufzug«, appear in the early 1740s. In order that we do not, as
often happens, consider all shorter plays without an act division to be one-act
plays, but can resort to a clearer terminology, the following criteria apply
to the database. We include plays that …

1. … are clearly marked as one-act plays in the subtitle.
2. … are described elsewhere as one-act plays (in *contemporary*
     sources like playbills, journals, etc.; example: Lessing's
     [»Philotas«](lessing-philotas)).
3. … are not operas or operettas.
4. … were published between 1740 and 1850.

A detailed reasoning for the composition of the corpus is presented
in Dîlan's dissertation.

## Technical Background

All data is collected in two simple YAML files, which can be downloaded
<a href="https://raw.githubusercontent.com/dracor-org/einakter/main/data.yaml">here</a>
(data.yaml) and
<a href="https://raw.githubusercontent.com/dracor-org/einakter/main/originals.yaml">here</a>
(originals.yaml).

We also export [JSON](https://einakter.dracor.org/data.json) and
[CSV](https://einakter.dracor.org/data.csv) files, which are
always based on the current version of the YAML files, but in contrast to the
source files they additionally contain some calculated values (currently the
number of characters per play). The JSON and CSV files can be easily accessed
from different environments to conduct statistics. An example in R:

```R
library(jsonlite)
einakter <- fromJSON("https://einakter.dracor.org/data.json")
hist(einakter$normalizedYear, breaks = 20, xlab="Jahrfünft", ylab="Anzahl Stücke im Korpus")
plot(einakter$normalizedYear, einakter$numberOfScenes, xlab="Jahr", ylab="Anzahl Szenen pro Stück")
plot(einakter$normalizedYear, einakter$numberOfCharacters, xlab="Jahr", ylab="Anzahl Figuren pro Stück")
```

The data model is described <a href="https://github.com/dracor-org/einakter/wiki">here</a>.
Versioning provided through our
[GitHub repository](https://github.com/dracor-org/einakter). The single-page app
is based on [React](https://reactjs.org/).

We also provide a [BEACON](https://meta.wikimedia.org/wiki/Dynamic_links_to_external_resources) file [here](https://einakter.dracor.org/wikidata.txt).
